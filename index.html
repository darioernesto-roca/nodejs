Node.js Cheat Sheet

1. Introduction

What is Node.js? Node.js is a cross-platform, open-source runtime environment that executes JavaScript code outside a browser. It's event-driven and non-blocking, built on Chrome's V8 engine.

Why use Node.js? It's fast, scalable, and ideal for real-time apps. It supports thousands of open-source packages via npm.

Node.js vs Browser:

  • Node: server-side, global, CommonJS, has file system access
  • Browser: client-side, window, ES6 modules, no file system

2. Modules

// CommonJS example
const foo = () => {
  console.log("Hello from foo!");
};
module.exports = foo;

const foo = require("./foo.js");
foo();
// ES Modules (in .mjs or with type: "module")
export const foo = () => console.log("Hello from foo!");
import { foo } from "./foo.mjs";
foo();

3. Built-in Modules

const fs = require("fs");
fs.writeFileSync("example.txt", "Hello, World!");

const http = require("http");
http.createServer((req, res) => {
  res.end("Hello World");
}).listen(3000);

const path = require("path");
console.log(path.join(__dirname, "file.txt"));

const os = require("os");
console.log(os.platform(), os.arch());

4. Error Handling

try {
  const result = riskyFunction();
} catch (err) {
  console.error(err);
}

process.on("uncaughtException", (err) => {
  console.error("Uncaught exception:", err);
});

5. Async Programming

// Callback
fs.readFile("file.txt", (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Promise
const readPromise = new Promise((resolve, reject) => {
  fs.readFile("file.txt", (err, data) => {
    if (err) reject(err);
    else resolve(data);
  });
});

// Async/Await
async function readFileAsync() {
  try {
    const data = await readPromise;
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
